# Overview #
Day 35 of 100 Days of Code made me feel like a true developer. 
Not necessarily because the code itself was incredibly challenging or even useful to others (as of now at least),
but rather because my OOP course in Java has taught me how to clean up my code for other programmers. 
Although I am not working on a team of software engineers, my use of Git and documentation was good practice for a
future in which I could collaborate with others.

In regard to Python, Day 35 taught me how to use API keys to fetch data that can be used to send alerts on WhatsApp.
The practicality of the project I found especially rewarding.

# Tech Stack #
- Python
- Requests
- Twilio
- OS Module

# Features #
- Requests a 12-hour forecast from OpenWeatherMap.
- Uses an API Key and API Token to complete the request.
- Can be changed depending on geographical position.
- Sends a message on WhatsApp about percipitation in the next 12 hours.

# What I Learned #
On top of learning how to use the various libraries from the tech stack, the most important feature I learned was how to use 
environment variables. Adding environment variables for privacy and organization pointed out to me the importance of protecting a person's 
API keys and phone numbers while still being able to publish material on GitHub. I also began applying concepts I learned from OOP,
such as documentations at the start of every function, showing people interested in the code how it works without needing to
understand every line.
